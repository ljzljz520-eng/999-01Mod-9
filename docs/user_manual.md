# FAQuery 用户手册

## 1. 系统简介
FAQuery 是一个轻量级的固定资产查询系统。它允许用户通过输入服务器 IP 和固定资产编码 (FACode)，快速获取对应的序列号 (SN)。系统支持多环境数据库切换，适用于同时也管理开发、测试及生产环境资产数据的场景。

## 2. 快速开始

### 访问系统
在浏览器中输入部署地址（通常为 http://localhost:3000）即可访问系统。

### 核心功能

#### 2.1 资产查询
1. **服务器 IP**: 输入目标数据库所在的服务器 IP 或域名（默认 `localhost` 指向内置数据库）。
2. **固定资产编码**: 输入需查询的 FACode（如 `FA001`）。
3. **查询**: 点击“查询 SN”按钮。
   - **成功**: 显示绿色的结果卡片，包含 SN 码。
   - **失败**: 显示红色的错误提示，包含具体原因（如“未找到”或“连接超时”）。

#### 2.2 历史记录
系统会自动记录最近的 20 条查询历史。
- **查看**: 首页下方“查询历史”区域。
- **重用**: 点击历史记录条目，自动填充 IP 和 FACode 到上方表单。
- **删除**: 点击条目右侧的垃圾桶图标可删除单条记录。
- **清空**: 点击标题栏右侧的“清空历史”可删除所有记录。

## 3. 高级功能：数据库连接管理

点击页面右上角的 **齿轮图标** 进入管理界面。

### 3.1 默认连接
系统内置一个“系统默认数据库 (MySQL)”，直接连接到 Docker 容器内的 MySQL 实例。该连接不可删除，确保系统总有可用数据源。

### 3.2 新增连接
如需查询外部数据库（如生产环境）：
1. 点击 **"+ 新增 MySQL 连接"**。
2. 填写连接信息：
   - **名称**: 给连接起个名字（如“生产库”）。
   - **主机/端口/用户/密码/数据库名**: 据实填写。
   - **快速填充**: 支持粘贴 `mysql://user:pass@host:port/dbname` 格式字符串，点击“解析”自动填表。
3. 点击 **创建连接**。
4. 在列表中点击 **启用**，即可切换当前查询的数据源。

> **⚠️ 注意**: 新增连接需配置 Public (公网) 可访问的生产环境数据库。

## 4. 常见问题排查

**Q: 点击查询没有反应？**
A: 请检查是否有红色的错误提示框。如果浏览器依然弹出英文气泡，请尝试强制刷新页面 (`Ctrl+F5`) 以更新到最新版本。

**Q: 只有 HTTP 400 错误？**
A: V6 版本已修复此问题，系统现在会显示具体的错误原因（如 "Access denied"）。请根据提示检查用户名密码是否正确。

**Q: 什么是 Public 数据库？**
A: 指可以通过互联网或当前网络环境直接访问的数据库。如果您的数据库在内网，请确保 FAQuery 部署服务器也在同一内网，或已配置 VPN/隧道。
