# 项目演练与验证记录 (Walkthrough)

**当前版本**: V6.4
**更新时间**: 2026-01-24

## 变更概要 (V6.4: 彻底消除原生验证气泡)

### 1. 问题复盘
即便使用了 `novalidate` 属性，部分浏览器场景下 `<button type="submit">` 仍可能触发原生的表单验证行为（英文气泡）。

### 2. 解决方案 (The Nuclear Fix)
- **机制变更**: 将查询表单和连接配置表单的提交按钮从 `type="submit"` 修改为 `type="button"`。
- **事件绑定**: 直接绑定 `onclick` 事件调用 JS 函数 (`window.queryManager.performQuery()`)。
- **效果**: 彻底切断了 Form 的 submit 事件流，浏览器不再有机会介入验证，所有逻辑完全由 JS 接管。

### 3. 部署状态
- **Rebuild**: 执行了增量构建，确保 `index.html` 的修改进入镜像。前端容器已重启。
- **状态**: 验证逻辑已完全由自定义 JS 控制。

## 验证指南 (手动测试)

1. **强制刷新页面** (`Ctrl+F5`)。
2. **查询验证**: 留空 FACode，点击“查询 SN”。
3. **新增连接验证**: 新增连接时留空必填项，点击“创建连接”。
4. **预期结果**: 无论如何操作，都不会再出现 "Please fill out this field" 气泡，只会弹出居中的中文 Alert 卡片。
